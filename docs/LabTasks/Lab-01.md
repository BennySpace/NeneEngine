## Задачи:
1. Класс Application. Реализовать класс Application, управляющий жизненным циклом
приложения, включая инициализацию, запуск основного цикла и завершение
программы.
2. Система управления состояниями игры. Разработать структуру для хранения и
смены различных игровых режимов (например, меню, игровое состояние, паузы),
используя паттерн "Состояния".
3. Расчёт дельта-времени (deltaTime). Организовать точную обработку временного
интервала между кадрами для обеспечения плавности изменений в игре (анимации,
физики и т.п.).
4. Система логирования. Настроить механизм ведения журнала для записи важных
событий и ошибок.
5. Aдаптер рендеринга (RenderAdapter). Создать интерфейс адаптера рендеринга,
позволяющий подключать различные графические API (OpenGL, DirectX, Vulkan и
др.) без внесения изменений в основной код.
6. Отрисовка тестовых примитивов. Через созданный адаптер реализовать
отображение простейшего геометрического элемента (треугольника или квадрата) в
графическом окне.

## Задания для самостоятельной работы:
1. Добавьте возможность поддержки нескольких окон одновременно.
2. Реализуйте изменение цвета фона окна через внешний конфиг (.json).
3. Если используете OpenGL или Vulkan, добавьте возможность горячего обновления
шейдеров.

## Дополнительное задание для дальнейшей работы над созданием игрового движка:
Добавьте в свою работу интеграцию ввода с клавиатуры и мыши, позволяющую
управлять положением тестового объекта на экране.

## Детали дополнительного задания:
1. Реализовать обработчик нажатия клавиш клавиатуры, позволяющий перемещать
тестовый примитив (треугольник или квадрат):
- стрелки вверх/вниз → вертикальное смещение,
- стрелки вправо/влево → горизонтальное смещение.
2. Обработчик кликов мышью:
- Левая кнопка мыши → масштабирует объект относительно центра экрана.
- Правая кнопка мыши → вращает объект вокруг своей оси.
3. Позиция и размеры объекта должны плавно меняться с учётом текущих значений
deltaTime.
4. Протестировать функциональность и убедиться, что движение гладкое и
соответствует ожиданиям.

## Важные рекомендации:
1. Используйте событие Input Events из среды разработки (Visual Studio, GLFW, SDL и
т.д.) для отслеживания действий игрока.
2. Обязательно учитывайте значение deltaTime при изменении позиции и размеров
объекта, чтобы обеспечить одинаковую скорость движения вне зависимости от FPS.
